データに名前つけできるハッシュを知ろう

ハッシュ(連想配列)複数のデータをまとめて扱うためのもの。

  • 配列との違いは、添字に数値ではなく文字列を利用できること。→データを名前で管理できる。
    • 配列だと・・・
  • [164,60]のように書けるがわかりにくい
  • 身長・体重にアクセスするのに添字が直感的ではない
    • ハッシュなら・・・
  • {height:164,weight:60}のようにわかりやすく書ける
  • 身長・体重にheightやweightの名前でアクセス出来る

    • ハッシュについて理解を深めよう

    ハッシュの基本的な記述方法

    'キー1'=> 値1,'キー2'=> 値2,・・・のようにキーと値のペアを記述して定義する。

    キー:ハッシュの値に名前付けする部分のこと

    ハッシュの要素を取得するにはキー名を使ってハッシュ['キー']と記述する。

    シンボルとは

    シンボル:プログラムで扱うデータに名前をつけるための仕組み。

    ハッシュのキーには文字列だけでなくシンボルも使える。

    シンボルは通常、先頭に:(コロン)をつけて定義する

    シンボルを用いると、ハッシュは{:key1 => 値1, :key => 値2,・・・}と記述できます。

    更に省略出来る→{ key1: 値1,key2:値2,・・・}となる。省略するときはキー名のシンボルの定義に必要な:(コロン)が先頭ではなくキー名の末尾に記述され、
    キー名と値の対応を表現する=>演算子が省略されている。

    例
    キーが文字列の場合

    {"key1" => 値1, "key2" => 値2,・・・}

    キーがシンボルの場合

    {:key1 => 値1, :key2 => 値2,・・・}

    キーがシンボルの場合(短縮形)

    {key1 :値1, key2 :値2,・・・}

    シンボルとは

    deleateメソッド：ハッシュの要素を削除する

    メソッドの引数にキー名を指定することで該当のキーと値のペアを削除できる。

    ハッシュ.delete(キー)

  • 配列のdeleteメソッドとほぼ同等の機能ですが、ハッシュのdeleteメソッドはハッシュ用のものであり、配列用に提供されるdeleteとは別物。
    • 例

    child = { height: 110, weight: 20, age:7 }

    #ハッシュの要素削除

    child.delete(:age)

    存在しないキー名を指定した場合

    ハッシュが格納された変数に存在しないキー名を指定した場合は、対応する要素も未定義なのでRubyで「無」を意味するnillが返却される。

    ハッシュに要素を追加するメソッド

    ハッシュ用のstoreメソッドでも要素を追加できる。

    storeメソッドは1番目の引数にキー名を、2番目の引数に値をカンマ(,)で区切って、ハッシュ.store(キー,値)と記述する。

    例

    ハッシュ.store(キー,値)

    child = {height: 110, weight: 20}

    child.store(:age, 7)
